SheetBridge API Reference

SheetBridge provides a unified REST interface over Google Sheets, Airtable, and Notion. Every record is normalized to the same schema regardless of the underlying source, so your application never needs to know which provider it is talking to.

Demo environment All examples use the demo key demo_key_sheetbridge. Data resets on each server restart. The demo server is running at the same origin as this page — try the live API now.

Authentication

Every request (except GET /health) must include an Authorization header with a Bearer token.

HTTP header
Authorization: Bearer demo_key_sheetbridge

If the token is missing or invalid, the API returns 401:

401 response
{
  "error": "Missing or invalid API key"
}

Base URL

All endpoints are served relative to the host:

Production
https://sheetbridge.io
Local / Replit dev
http://localhost:3000

GET /health

No authentication required. Use this endpoint for uptime monitoring and to verify the API version.

GET /health No auth required
curl example
curl https://sheetbridge.io/health
200 response
{
  "status":    "ok",
  "version":   "1.0.0",
  "timestamp": "2026-03-28T14:00:00.000Z"
}

GET /v1/records

Returns a paginated list of records. Supports optional filtering, sorting, and cursor-based pagination.

GET /v1/records

Query parameters

ParameterTypeDescriptionExample
sourcestringFilter by source typesheets
filterstringField filter expression (FieldName:Value)Status:Active
sortstringSort expression (FieldName:asc|desc)Revenue:desc
limitintegerRecords per page (1–100, default 10)25
cursorstringBase64 pagination cursor from previous responseeyJvZmZzZXQiOjEwfQ==
curl example
curl "https://sheetbridge.io/v1/records?filter=Status:Active&sort=Revenue:desc" \
  -H "Authorization: Bearer demo_key_sheetbridge"
200 response
{
  "records": [
    {
      "id":         "row_001",
      "source":     "sheets",
      "fields":     { "Name": "Acme Corp", "Status": "Active", "Revenue": 12000 },
      "created_at": "2026-03-01T10:00:00Z",
      "updated_at": "2026-03-01T10:00:00Z"
    }
  ],
  "next_cursor": null,
  "total": 3
}

GET /v1/records/:id

Returns a single record by its ID. Returns 404 if not found.

GET /v1/records/:id
curl example
curl https://sheetbridge.io/v1/records/row_001 \
  -H "Authorization: Bearer demo_key_sheetbridge"
200 response
{
  "id":         "row_001",
  "source":     "sheets",
  "fields":     { "Name": "Acme Corp", "Status": "Active", "Revenue": 12000 },
  "created_at": "2026-03-01T10:00:00Z",
  "updated_at": "2026-03-01T10:00:00Z"
}
404 response
{ "error": "Record not found", "id": "row_999" }

POST /v1/records

Creates a new record. An ID is generated automatically. Returns 201 with the created record.

POST /v1/records

Request body

FieldTypeRequiredDescription
sourcestringYesSource type: sheets, airtable, or notion
fieldsobjectYesArbitrary key-value pairs representing the row data
curl example
curl -X POST https://sheetbridge.io/v1/records \
  -H "Authorization: Bearer demo_key_sheetbridge" \
  -H "Content-Type: application/json" \
  -d '{"source":"sheets","fields":{"Name":"Zeta Corp","Status":"Trial","Revenue":0}}'
201 response
{
  "id":         "row_a3f9c12b",
  "source":     "sheets",
  "fields":     { "Name": "Zeta Corp", "Status": "Trial", "Revenue": 0 },
  "created_at": "2026-03-28T14:05:00.000Z",
  "updated_at": "2026-03-28T14:05:00.000Z"
}

PATCH /v1/records/:id

Merges the provided fields into the existing record. Fields not mentioned in the request body are preserved unchanged. Updates updated_at automatically.

PATCH /v1/records/:id
curl example
curl -X PATCH https://sheetbridge.io/v1/records/row_002 \
  -H "Authorization: Bearer demo_key_sheetbridge" \
  -H "Content-Type: application/json" \
  -d '{"fields":{"Status":"Active","Revenue":5000}}'
200 response
{
  "id":         "row_002",
  "source":     "sheets",
  "fields":     { "Name": "Beta Labs", "Status": "Active", "Revenue": 5000 },
  "created_at": "2026-03-05T14:00:00Z",
  "updated_at": "2026-03-28T14:06:00.000Z"
}

DELETE /v1/records/:id

Permanently removes a record from the data store. This action cannot be undone in the demo environment.

DELETE /v1/records/:id
curl example
curl -X DELETE https://sheetbridge.io/v1/records/row_004 \
  -H "Authorization: Bearer demo_key_sheetbridge"
200 response
{ "deleted": true, "id": "row_004" }

GET /v1/sources

Returns all configured data sources and their connection status.

GET /v1/sources
curl example
curl https://sheetbridge.io/v1/sources \
  -H "Authorization: Bearer demo_key_sheetbridge"
200 response
{
  "sources": [
    { "id": "src_001", "type": "sheets",   "name": "Customer CRM Sheet",    "status": "active"      },
    { "id": "src_002", "type": "airtable", "name": "Product Backlog Base",   "status": "coming_soon" },
    { "id": "src_003", "type": "notion",   "name": "Company Wiki DB",       "status": "coming_soon" }
  ]
}

Pagination

SheetBridge uses cursor-based pagination. The cursor is a base64-encoded JSON object containing the current offset. When next_cursor is null, you have reached the last page.

Cursor structure (decoded)
{ "offset": 10 }
Fetching the next page
curl "https://sheetbridge.io/v1/records?limit=2&cursor=eyJvZmZzZXQiOjJ9" \
  -H "Authorization: Bearer demo_key_sheetbridge"
Stability guarantee Cursors are stable as long as the underlying dataset does not change between pages. In the demo, new records may be inserted by other users — if you need strict consistency, filter by source to narrow the result set.

Filtering & Sorting

Both filter and sort use a simple colon-separated syntax.

ParameterFormatExampleNotes
filterFieldName:ValueStatus:ActiveCase-insensitive string match; numeric equality for number fields
sortFieldName:asc or FieldName:descRevenue:descNumeric sort for numeric fields; lexicographic otherwise

Error Codes

StatusMeaningWhen it occurs
400Bad RequestMissing required fields (source, fields) or malformed JSON body
401UnauthorizedMissing or invalid Authorization: Bearer header
404Not FoundRecord ID does not exist or endpoint path is incorrect
429Too Many RequestsRate limit exceeded — back off and retry after the Retry-After header value
500Internal Server ErrorUnexpected server-side error — please open an issue on GitHub

All error responses share the same JSON shape:

Error response shape
{
  "error": "Human-readable message explaining the problem"
}

SDKs & Client Libraries

Official SDKs are in development. In the meantime:

  • Download openapi.json and use openapi-generator to produce typed clients for TypeScript, Python, Go, Java, and more.
  • Import the spec into Postman or Insomnia to explore all endpoints interactively.
  • The API is fully compatible with any HTTP client — fetch, axios, httpx, requests, etc.
